# Discord-бот «Скомпоновать»

Бот собирает все фотографии из канала (где вызвана команда) и отправляет их в канал **result** пачками по 10 изображений в одном сообщении.

- **Application ID:** `1478116662468284517`
- **Команды:** `/скомпоновать` — собрать фото в result; `/посчитать_скрины` — сколько скринов/фото в этом канале; `/очистить_result` — удалить все сообщения в канале result
- **Канал с фото:** любой (название не важно), команда вызывается в том канале, откуда нужно собрать фото.
- **Канал для результата:** должен называться **result** (создайте его на сервере вручную).

---

## Как добавить бота на сервер

### 1. Токен бота (обязательно)

В Discord нужен **Bot Token**, а не только Application ID и Public Key.

1. Откройте [Discord Developer Portal](https://discord.com/developers/applications).
2. Выберите своё приложение (или создайте новое).
3. Слева откройте **Bot**.
4. Нажмите **Reset Token** или **Copy** и скопируйте **токен**.  
   Храните его в секрете и никому не показывайте.

### 2. Приглашение бота на сервер

1. В Developer Portal откройте **OAuth2** → **URL Generator**.
2. В **SCOPES** отметьте **bot**.
3. В **BOT PERMISSIONS** отметьте:
   - Read Messages/View Channels
   - Send Messages
   - Read Message History
   - Attach Files
   - Manage Messages (нужно для команды очистки канала result)
   - Use Slash Commands
4. Скопируйте сгенерированную ссылку внизу страницы.
5. Откройте её в браузере, выберите сервер и нажмите **Authorize**.

После этого бот появится на сервере в списке участников (оффлайн до запуска скрипта).

### 3. Message Content Intent (если бот не видит картинки)

Если команда `/скомпоновать` пишет «не найдено ни одного изображения», хотя в канале есть фото:

1. В Developer Portal откройте **Bot** (левое меню).
2. Пролистайте до **Privileged Gateway Intents**.
3. Включите **Message Content Intent** (переключатель в положение ON).
4. Сохраните изменения и перезапустите бота.

У бота также есть запасной вариант: при пустом результате он запрашивает сообщения напрямую через API (последние 100 сообщений канала).

### 4. Канал для результата

На сервере создайте текстовый канал с именем **result**. В него бот будет отправлять скомпонованные фото.

## Как запустить бота

### Требования

- Python 3.10 или новее
- Токен бота из пункта «Токен бота» выше

### Установка зависимостей

В папке проекта:

```bash
# Windows (PowerShell)
.\.venv\Scripts\activate
pip install -r requirements.txt

# или без venv
pip install -r requirements.txt
```

### Запуск

**Вариант 1 — файл .env (удобно)**

1. Скопируйте `.env.example` в `.env`:  
   `copy .env.example .env`
2. Откройте `.env` и вставьте свой токен:  
   `DISCORD_BOT_TOKEN=ваш_токен_из_раздела_Bot`
3. Запустите:  
   `python main.py`

Файл `.env` не коммитьте в git (в нём секрет).

**Чтобы слэш-команды появлялись сразу** (в т.ч. вторая команда `/очистить_result`), добавьте в `.env` ID своего сервера:
```env
DISCORD_GUILD_ID=123456789012345678
```
Как узнать ID: Discord → Настройки пользователя → Расширенные → включите **Режим разработчика**. Затем правый клик по названию сервера в списке слева → **Копировать ID сервера**. Вставьте это число в `.env` как `DISCORD_GUILD_ID`. Перезапустите бота — обе команды появятся на сервере сразу. Без `DISCORD_GUILD_ID` новые команды могут отображаться в Discord до часа.

**Вариант 2 — переменная окружения**

```powershell
$env:DISCORD_BOT_TOKEN = "ВАШ_ТОКЕН_БОТА"
python main.py
```

После запуска в консоли должно появиться что-то вроде:  
`Бот запущен: ИмяБота#1234 (ID: ...)` и `Синхронизировано команд: 1`.

## Использование

1. Убедитесь, что в канале с фото (например, «отчет-повыха-8-9») есть сообщения с изображениями.
2. В **этом же** канале введите команду `/скомпоновать` и отправьте.
3. Бот прочитает до 1000 последних сообщений, соберёт все изображения по порядку (от старых к новым) и отправит их в канал **result** по 10 фото в каждом сообщении.

Если канала **result** нет или у бота нет прав на чтение/запись — бот напишет об этом в ответ на команду.
